# D-20260721-001 · VNW→AIT首个试点节点选择

> 对应矩阵v5.0"决策与来源"节的D1（"VNW→AIT首个节点：选择已有SOP和真实案例的节点子集；是否以COM为MVP"）；同时解决Agent体系搭建项目章程v1.1的开放问题R8。

---

## 决策条目

| 字段 | 内容 |
|------|------|
| **决策时间** | 2026-07-21 |
| **决策场景** | 架构v1.3把AIT重新定义为VNW流水线的下游延伸段（VNW产出SOP即触发AIT介入），不必等VNW全域覆盖完成，可拿"已推动到SOP阶段的节点子集"做MVP——需要选定具体是哪个节点 |
| **决策问题** | VNW→AIT打通的第一个真实试点，选哪个节点？ |
| **可选方案** | L3-COM（佣金全链路管理，已有8份手工分析文档）/ 其他PAY域已验证节点（VNW Phase1真实验证过PAY域9节点、4个熔断节点）/ 暂不选，等VNW基线合并批次完成后再定 |
| **最终选择** | L3-COM |
| **选择理由** | L3-COM是目前唯一一个完整走过"SOP→AI化设计→分期搭建"全过程的真实案例（架构v1.3 §5.4已用它反推AIT设计框架），四维就绪度矩阵（R/D/P/O）、Tier分类、三级串联验证链、资金风险覆盖原则等关键设计都已经在这个案例里校准过一遍——用它做AIT的第一个真实试点，是把"已经手工验证过的设计"接上"自动化实现"，而不是从一个全新节点重新摸索，降低了MVP阶段的不确定性 |
| **承担风险** | L3-COM是Jasper手工分析产出，不是VNW自动跑出来的，AIT试点阶段可能需要先把L3-COM现有的8份文档"倒推"成VNW该产出的标准格式（节点锚点+规则级溯源SOP），这一步本身没有被验证过，可能比预想的更费时间；另外L3-COM验证出的规则（如资金风险覆盖原则）是否能代表其他节点的普遍情况，需要在后续第二个节点上做交叉验证，不能只用一个样本就下结论 |
| **验证方式** | AIT能否基于L3-COM的既有材料，产出跟8份手工文档结论一致的三轨道拆解（机器规则/人的规则/协同新规则）；对比手工版和AIT版在Human gate判定（尤其资金风险覆盖原则）上是否一致 |
| **决策人** | Jasper |
