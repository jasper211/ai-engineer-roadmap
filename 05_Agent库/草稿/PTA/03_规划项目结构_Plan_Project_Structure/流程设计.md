## 二、每个 L3 流程详细设计

### L3-PTA-01 · 任务解析

**回答问题**：用户想要什么？

```
用户指令 ──→ PTA-S01 意图解析器 ──→ 结构化任务包
              ↓
         判断任务类型
              ↓
    ┌─────────┼─────────┐
    ▼         ▼         ▼
 通用任务   价值节点任务  转型咨询任务
    ↓         ↓         ↓
  PTA-S02  调用 VNW    调用 AIT
```

**输入**：自然语言指令（如"按顺序完成 P0-02, P0-03, P1-03, P1-04"）
**输出**：结构化任务包（JSON）

```json
{
  "task_id": "T-20260703-001",
  "type": "sequential",  // sequential / parallel / conditional
  "priority": "P1",
  "items": [
    {"id": "P0-02", "name": "前端 Vercel 部署", "status": "pending"},
    {"id": "P0-03", "name": "MCP Server 公开", "status": "pending"},
    {"id": "P1-03", "name": "信号提取扩展到全量", "status": "pending"},
    {"id": "P1-04", "name": "访谈规则继承", "status": "pending"}
  ],
  "constraints": ["按顺序", "不跳过"],
  "context": "Phase 1 收尾任务"
}
```

**子 Agent**：PTA-S01 意图解析器

---

### L3-PTA-02 · 执行编排

**回答问题**：怎么执行？

```
任务包 ──→ PTA-S02 执行调度器 ──→ 执行计划
              ↓
    ┌─────────┼─────────┐
    ▼         ▼         ▼
 步骤1     步骤2     步骤3
    ↓         ↓         ↓
 工具A     工具B     工具C
```

**输入**：结构化任务包（JSON）
**输出**：执行计划（步骤清单 + 工具调度）

```json
{
  "plan_id": "P-20260703-001",
  "steps": [
    {
      "seq": 1,
      "action": "deploy_vercel",
      "tool": "bash",
      "command": "cd app_v2 && vercel --prod",
      "depends_on": [],
      "timeout": 300
    },
    {
      "seq": 2,
      "action": "push_mcp",
      "tool": "bash",
      "command": "cd process-db-mcp && git push",
      "depends_on": [1],
      "timeout": 60
    }
  ]
}
```

**子 Agent**：PTA-S02 执行调度器

---

### L3-PTA-03 · 进度监控

**回答问题**：执行到哪了？有没有问题？

```
执行计划 ──→ PTA-S03 进度追踪器 ──→ 进度报告
                ↓
           检测异常
                ↓
         ┌──────┴──────┐
         ▼             ▼
       正常           异常
         ↓             ↓
      继续执行      通知用户
                      ↓
                   等待决策
```

**输入**：执行步骤状态
**输出**：进度报告 + 风险预警

```json
{
  "report_time": "2026-07-03T16:00:00",
  "overall_status": "in_progress",  // pending / in_progress / completed / failed
  "completed_steps": 2,
  "total_steps": 5,
  "current_step": {
    "seq": 3,
    "action": "generate_baseline",
    "status": "running",
    "start_time": "2026-07-03T15:55:00",
    "estimated_end": "2026-07-03T16:05:00"
  },
  "alerts": []
}
```

**子 Agent**：PTA-S03 进度追踪器

---

### L3-PTA-04 · 产出同步

**回答问题**：任务完成了，文档更新了吗？

```
任务完成信号 ──→ PTA-S04 文档同步器 ──→ 同步完成
                    ↓
           ┌────────┼────────┐
           ▼        ▼        ▼
        Git提交   看板更新  执行记录
```

**输入**：任务完成信号（JSON）
**输出**：Git 提交 + 看板更新 + 执行记录

**同步清单**：
- [ ] 更新 `能力整改看板.md`（进度、状态）
- [ ] 创建/更新 `任务执行记录.md`
- [ ] Git add + commit + push
- [ ] 更新 `PhaseX_完成总结.md`（如需要）

**子 Agent**：PTA-S04 文档同步器

---

### L3-PTA-05 · 归档复盘

**回答问题**：这次任务有什么经验教训？

```
任务历史 ──→ PTA-S05 归档复盘器 ──→ 执行记录 + 教训库
                ↓
           ┌────┴────┐
           ▼         ▼
        成功经验   踩坑记录
           ↓         ↓
        最佳实践   F3 教训库
```

**输入**：任务完整历史（指令 + 执行日志 + 产出）
**输出**：执行记录.md + 教训库条目

**复盘问题清单**：
1. 任务理解是否准确？（有无偏差？）
2. 执行过程是否顺利？（有无阻塞？）
3. 产出是否符合预期？（有无遗漏？）
4. 有什么可以改进的？（工具/流程/沟通）

**子 Agent**：PTA-S05 归档复盘器

## 三、子 Agent 接口契约

### PTA-S01 意图解析器

| 字段 | 说明 |
|------|------|
| **输入** | `user_input: str`（自然语言指令） |
| **输出** | `TaskPackage: dict`（结构化任务包） |
| **错误处理** | 指令模糊时，返回澄清问题列表 |

### PTA-S02 执行调度器

| 字段 | 说明 |
|------|------|
| **输入** | `task_package: dict`（结构化任务包） |
| **输出** | `ExecutionPlan: dict`（执行计划） |
| **错误处理** | 任务包格式错误时，返回错误信息 |

### PTA-S03 进度追踪器

| 字段 | 说明 |
|------|------|
| **输入** | `execution_plan: dict` + `current_status: dict` |
| **输出** | `ProgressReport: dict`（进度报告） |
| **错误处理** | 步骤超时/失败时，返回异常报告 |

### PTA-S04 文档同步器

| 字段 | 说明 |
|------|------|
| **输入** | `task_completion: dict`（任务完成信号） |
| **输出** | `SyncResult: dict`（同步结果） |
| **错误处理** | Git 提交失败时，返回错误 + 手动操作指南 |

### PTA-S05 归档复盘器

| 字段 | 说明 |
|------|------|
| **输入** | `task_history: dict`（任务完整历史） |
| **输出** | `ReviewReport: dict`（复盘报告） |
| **错误处理** | 历史数据缺失时，返回警告 |
