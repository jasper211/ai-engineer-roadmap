## 一、设计背景

### 1.1 为什么需要 Agent 体系

Phase 1 完成了多个自动化脚本（信号提取、规则合并、一致性校验），但存在以下问题：

| 问题 | 表现 | Agent 化解决 |
|------|------|-------------|
| 脚本分散 | 每个脚本独立运行，无统一调度 | 主 Agent 统一编排 |
| 触发被动 | 需要用户手动执行 | 子 Agent 主动监控 + 触发 |
| 产出孤立 | 信号基线、规则空白、看板更新各自为政 | 端到端自动同步 |
| 认知断层 | 每次新对话需重新理解上下文 | Agent 配置持久化上下文 |

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **L3 端到端** | 每个主 Agent 对应一个完整的 L3 流程，子 Agent 对应 L4/L5 活动 |
| **单一职责** | 主 Agent 不跨域，PTA 管项目、VNW 管价值节点、AIT 管转型咨询 |
| **可组合** | 子 Agent 可被多个主 Agent 复用（如 PTA-S04 文档同步器可被 VNW 调用） |
| **渐进实现** | 先实现 VNW（业务价值最直接），再 PTA（通用性最强），最后 AIT（战略价值） |

## 二、总体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    AI 协同中枢层                              │
│         （任务路由、状态管理、跨 Agent 协调）                 │
└─────────────────────────────────────────────────────────────┘
        │              │              │
        ▼              ▼              ▼
┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│  主 Agent 1   │ │  主 Agent 2   │ │  主 Agent 3   │
│  项目任务协同  │ │ 价值节点驱动  │ │  AI协同转型   │
│   (PTA)      │ │  工作流 (VNW) │ │   (AIT)      │
└──────────────┘ └──────────────┘ └──────────────┘
```

### 2.1 职责边界

| 维度 | PTA | VNW | AIT |
|------|-----|-----|-----|
| **服务对象** | 任意项目/任务 | 价值节点分析工作流 | 协同部门 AI 转型 |
| **核心问题** | "这个任务怎么做？" | "这个域的规则空白是什么？" | "这个部门怎么 AI 化？" |
| **输入** | 自然语言指令 | 价值节点清单 + 流程蓝图 | 部门访谈 + 流程文档 |
| **输出** | 执行记录 + 归档文档 | 信号基线 + 规则空白地图 | AI 方案 + Agent + SOP |
| **触发频率** | 每次任务 | 清单更新/新域开展 | 按需咨询 |

## 三、主 Agent 1：项目任务协同 Agent (PTA)

### 3.1 定位

通用型项目执行引擎，适用于任何项目的任务全生命周期管理。

### 3.2 L3 端到端流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ L3-PTA-01│ → │ L3-PTA-02│ → │ L3-PTA-03│ → │ L3-PTA-04│ → │ L3-PTA-05│
│ 任务解析 │    │ 执行编排 │    │ 进度监控 │    │ 产出同步 │    │ 归档复盘 │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
```

### 3.3 子 Agent 设计

| 子 Agent | 职责 | 输入 | 输出 | 对应现有能力 |
|----------|------|------|------|------------|
| **PTA-S01 意图解析器** | 理解用户指令，识别任务类型、优先级、约束条件 | 自然语言指令 | 结构化任务包（JSON） | 当前由 AI 手动完成 |
| **PTA-S02 执行调度器** | 分解任务为可执行步骤，调度工具（Python/browser-use/Git） | 任务包 | 执行计划 + 步骤清单 | 当前手动规划 |
| **PTA-S03 进度追踪器** | 监控任务执行状态，处理阻塞/异常，向用户汇报 | 执行步骤状态 | 进度报告 + 风险预警 | 当前手动更新看板 |
| **PTA-S04 文档同步器** | 任务完成后自动更新看板、执行记录、README、Git 提交 | 任务完成信号 | git commit + 看板更新 | 当前手动操作 |
| **PTA-S05 归档复盘器** | 生成执行记录、沉淀经验教训、更新 F3 教训库 | 任务完整历史 | 执行记录.md + 教训库条目 | 当前手动撰写 |

### 3.4 触发场景

| 用户指令 | PTA 响应 |
|---------|---------|
| "按顺序完成 P0-02, P0-03, P1-03, P1-04" | 解析任务列表 → 按序执行 → 每步同步看板 |
| "回顾下进度，继续推进" | 读取看板 → 识别已完成/待启动 → 推荐下一步 |
| "方向不对，这个前端是草稿" | 认知修正 → 更新理解 → 重新规划 |
| "执行，对的都在" | 确认执行条件 → 开始执行 → 实时汇报 |

## 四、主 Agent 2：价值节点驱动工作流 Agent (VNW)

### 4.1 定位

业务型工作流引擎，专门处理价值节点 → 信号 → 规则空白 → SOP 的完整链路。

### 4.2 L3 端到端流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ L3-VNW-01│ → │ L3-VNW-02│ → │ L3-VNW-03│ → │ L3-VNW-04│ → │ L3-VNW-05│
│ 节点解析 │    │ 信号提取 │    │ 基线合并 │    │ 规则空白 │    │ SOP底稿 │
│         │    │         │    │         │    │ 地图生成 │    │ 生成   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     ↑                                                              ↓
     └──────────────────────────────────────────────────────────────┘
                         反馈闭环（清单更新后重新触发）
```

### 4.3 子 Agent 设计

| 子 Agent | 职责 | 输入 | 输出 | 对应现有脚本 |
|----------|------|------|------|------------|
| **VNW-S01 清单监控器** | 监控价值节点清单 Excel 变更（mtime/哈希） | 文件系统事件 | 变更通知 + 变更摘要 | 无（需新建） |
| **VNW-S02 节点解析器** | 读取 Excel，提取节点编码、名称、域归属、物理对应 | 清单 Excel | 结构化节点数据（JSON） | extract_signals.py 部分 |
| **VNW-S03 信号提取器** | 基于节点数据，按三层递进法提取 7 类信号 | 节点数据 + 蓝图 | 信号基线（Markdown） | extract_signals.py |
| **VNW-S04 基线合并器** | 合并自动基线与手工基线（访谈规则继承） | auto_vX.md + manual_vY.md | 完整基线 auto_vZ.md | merge_interview_rules.py |
| **VNW-S05 规则空白生成器** | 分析基线，识别 C 类行动项、熔断节点、信号缺失 | 完整基线 | 规则空白地图（HTML） | 刚实现的 HTML 生成 |
| **VNW-S06 SOP底稿生成器** | 基于 A 类规则 + 流程蓝图，生成 SOP 初稿 | 基线 + 蓝图 | SOP Markdown 草稿 | 无（未来扩展） |
| **VNW-S07 校验验证器** | 双文档交叉验证（清单 vs 蓝图），输出一致性报告 | 清单 + 蓝图 | 校验报告（HTML） | validate_v2_all_blueprints.py |

### 4.4 触发场景

| 事件 | VNW 响应 |
|------|---------|
| 价值节点清单原表更新 | 监控 → 标准化 → 信号提取 → 基线生成 → 规则空白更新 |
| 新域开展规则分析 | 读取蓝图 → 节点解析 → 信号提取 → 校验 → 基线 |
| 访谈完成，手工基线更新 | 读取 manual → 合并 → 更新规则空白 |
| 用户询问"某域规则空白" | 读取当前基线 → 生成该域规则空白预览 |

## 五、主 Agent 3：AI 协同转型 Agent (AIT)

### 5.1 定位

战略型咨询引擎，帮助协同部门识别 AI 化机会、设计流程、构建 Agent、交付 SOP。

### 5.2 L3 端到端流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ L3-AIT-01│ → │ L3-AIT-02│ → │ L3-AIT-03│ → │ L3-AIT-04│ → │ L3-AIT-05│
│ 需求识别 │    │ 流程分析 │    │ AI化设计 │    │ Agent构建 │    │ SOP交付 │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
```

### 5.3 子 Agent 设计

| 子 Agent | 职责 | 输入 | 输出 | 说明 |
|----------|------|------|------|------|
| **AIT-S01 需求识别器** | 访谈部门负责人，识别痛点、高频重复工作、信息孤岛 | 访谈记录/会议纪要 | 需求清单（优先级排序） | 基于三层递进法访谈框架 |
| **AIT-S02 流程分析器** | 梳理现有流程（L3-L5），识别可 AI 化环节 | 流程文档/SOP/访谈 | 流程图 + AI 化评估矩阵 | 输出类似 Agent 能力评估 |
| **AIT-S03 AI 化设计师** | 设计 AI 介入方案（全自动/人机协作/辅助） | 流程分析结果 | AI 方案设计文档 | 含工具选型、模型建议 |
| **AIT-S04 Agent 构建器** | 基于设计方案，构建具体 Agent（配置 + 脚本） | AI 方案设计 | Agent JSON 配置 + Python 脚本 | 复用 PTA/VNW 的子 Agent |
| **AIT-S05 SOP 生成器** | 将 AI 方案转化为可执行的 SOP 文档 | Agent 设计 + 流程 | SOP 手册（Markdown） | 含人机协作边界、异常处理 |
| **AIT-S06 效果验证器** | 部署后监控效果，收集反馈，迭代优化 | 运行日志 + 用户反馈 | 效果报告 + 迭代建议 | 闭环改进 |

### 5.4 触发场景

| 用户指令 | AIT 响应 |
|---------|---------|
| "财务部门想引入 AI 辅助对账" | 需求访谈 → 流程分析 → AI 方案 → Agent 构建 → SOP |
| "帮我们设计一个佣金结算的 AI 工作流" | 识别现有流程 → 提取规则 → 设计 Agent → 交付 |
| "这个岗位哪些工作可以 AI 替代" | 岗位分析 → 活动清单 → AI 评估 → 方案设计 |

## 七、实现路线图

### 7.1 优先级排序

| 优先级 | 主 Agent | 理由 | 预计工作量 |
|--------|---------|------|-----------|
| P1 | **VNW** | 与当前工作流最紧密，已有脚本可封装，业务价值最直接 | 2-3 周 |
| P2 | **PTA** | 通用性强，可立即提升协同效率，为 VNW/AIT 提供执行框架 | 1-2 周 |
| P3 | **AIT** | 战略价值高，但依赖 VNW 成熟（需信号提取能力） | 3-4 周 |

### 7.2 VNW 实现计划（P1）

| 周次 | 任务 | 产出 |
|------|------|------|
| W1 | VNW-S01 清单监控器 + VNW-S02 节点解析器 | 监控脚本 + 节点解析 JSON |
| W2 | VNW-S03 信号提取器封装 + VNW-S04 基线合并器 | 可配置的信号提取 Agent |
| W3 | VNW-S05 规则空白生成器 + VNW-S07 校验验证器 | 自动化规则空白地图生成 |
| W4 | 集成测试 + 文档同步 | VNW v1.0 发布 |

### 7.3 关键技术选型

| 组件 | 技术 | 理由 |
|------|------|------|
| 文件监控 | watchdog (Python) | 跨平台，稳定 |
| 任务调度 | cron + Python | 简单，易维护 |
| 配置管理 | JSON + Pydantic | 类型安全，易验证 |
| 通知推送 | 企业微信 Webhook | 已有使用经验 |
| 版本控制 | Git + Python 调用 | 已有使用经验 |
