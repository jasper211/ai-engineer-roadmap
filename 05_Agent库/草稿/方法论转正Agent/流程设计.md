## 二、收集过滤机制 v0（人工规则，不追求自动化）

判断一条 AI 协同方法论相关的行业信息"是否值得落成源文档"，先用最简单的人工判断规则：

1. **相关性**：是否直接涉及 AI/Agent 协同、人机分工、Agent 验证方法论这几个我们已有方法论笔记覆盖的主题（可对照 vault 内 L1-L16 系列笔记判断是否已有覆盖/是否补充新视角）
2. **来源可信度**：优先选有具体来源（公司/团队/论文/知名从业者）的一手信息，避免转载多手、来源不明的二手内容
3. **是否有新增判断价值**：只是重复我们已知的内容不收录；提供了新的判断依据、反例、或验证方法的才收录

不满足以上任一条的，先不落地，不强行凑数。这套规则本身也是待验证的草案，后续按实际筛选经验调整。

## 三、md 源文档的 schema（草案）

落地的每份源文档需要在 frontmatter 挂载以下字段（定义详见 vault 内 `方法论知识库/行业学习/_schema定义.md`，2026-07-23 该目录已从嵌套在 `Jasper AI协同经验引擎/` 下迁移到 vault 顶层，跟 OBAgent 已建的自动化白名单排除区对齐）：

- `source_url`
- `collected_at`
- `staleness_review_date`
- `info_type`
- `evidence_basis`
- `entity_type`/`entity_ref`（可选，预留——若未来接入 OB 现有 entity_ref 枢纽机制才需要，早期原型阶段可留空）

这套 schema **独立于 OB 现有五字段**（type/concept_type/project/source/extracted_at），因为内部一手资料与外部二手信息的可信度/时效语义不同（见《行业自学习线_启动与执行方案》二节结论4）。
