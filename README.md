## 🎯 这个项目做什么

对照一份年薪 50-70 万的 AI 算法与工程岗位标准，以「懂业务、懂方法论的 AI 工程师」为差异化定位，识别出 11 个能力维度，按 P0/P1/P2/P3 优先级分 4 个阶段补齐差距：

| 阶段 | 周期 | 核心目标 | 关键产出 |
|------|------|---------|---------|
| **Phase 0** | 第 0 周 | 启动准备 | GitHub repo、看板、决策日志、周报模板 |
| **Phase 1** | 第 1-4 周 | 补齐硬缺口 + 推上线 | Computer Use 实操报告、前端 App 上线、MCP Server 公开 |
| **Phase 2** | 第 5-8 周 | Agent 产品化 | B-RPT Agent 上线、模型选型矩阵、Skill 公开发布 |
| **Phase 3** | 第 9-12 周 | 系统性与扩展性 | AI 技术雷达、飞书集成、成本优化报告 |
| **Phase 4** | 第 13-16 周 | 长期补足 + 作品集 | Hermes 试用、技术文章、个人作品集网站 |

**核心逻辑**：把"内部验证"变成"外部可见"，把"设计文档"变成"运行服务"。

## 📊 能力维度跟踪

| 维度 | 招聘标准 | 当前状态 | 目标 | 负责阶段 | 进度 |
|------|---------|---------|------|---------|------|

| 大模型使用与边界 | Claude / GPT / Gemini | 部分覆盖 | 3+ 模型 + 选型矩阵 | P2 | 55% |
| 全流程独立交付 | 需求拆解 → 架构设计 → 上线 | 较强 | 前端上线 + MCP 公开 + Agent 独立运行 | P1 | 75% |
| 技术嗅觉 | 跟踪新模型/框架/协议 | 有一定基础 | 技术雷达 + 前沿框架 demo | P3 | 45% |
| 工程落地效率 | 封装 Agent/Skills 到生产场景 | 部分覆盖 | 公开包 + 量化效率数据 | P2 | 55% |

- **数据库**：SQLite / PostgreSQL (via Metabase)
