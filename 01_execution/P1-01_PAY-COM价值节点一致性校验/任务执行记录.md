## 四、执行方案

### 4.1 技术路线

采用 **Python 脚本 + browser-use MCP 混合方案**：

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  流程蓝图.md    │     │  价值节点清单   │     │  browser-use    │
│  (Markdown)     │     │  .xlsx          │     │  MCP            │
└────────┬────────┘     └────────┬────────┘     └────────┬────────┘
         │                       │                       │
         └───────────────────────┼───────────────────────┘
                                 ↓
                    ┌─────────────────────┐
                    │   Python 解析脚本    │
                    │  · 提取 VN 编码      │
                    │  · 提取 VN 名称      │
                    │  · 比对差异          │
                    └──────────┬──────────┘
                               ↓
                    ┌─────────────────────┐
                    │  生成本地 HTML 报告  │
                    └──────────┬──────────┘
                               ↓
                    ┌─────────────────────┐
                    │ browser-use 截图    │
                    │  · 打开 HTML 报告   │
                    │  · 全页截图保存     │
                    └─────────────────────┘
```

**为什么不用纯 browser-use？**
- Excel 文件直接解析用 Python 更稳定
- Markdown 表格用正则/解析器提取比 OCR 更可靠
- browser-use 负责「可视化验证」和「截图证据」，这是它的强项

**最佳实践：让合适的工具做合适的事。**

### 4.2 步骤拆解

| 步骤 | 动作 | 工具 | 输出 |
|------|------|------|------|
| 1 | 解析 Markdown 提取 VN 表格 | Python + markdown/re | `com_blueprint_vn.json` |
| 2 | 解析 Excel 提取 VN 总览 | Python + pandas | `d1_vn_overview.json` |
| 3 | 执行一致性比对 | Python | `validation_report.json` |
| 4 | 生成 HTML 报告 | Python + Jinja2/simple HTML | `validation_report.html` |
| 5 | 用 browser-use 打开报告并截图 | browser-use MCP | `report_screenshot.png` |
| 6 | 生成 Markdown 执行总结 | 手动/AI 辅助 | 本文件附录 |
