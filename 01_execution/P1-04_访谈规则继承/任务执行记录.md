## 二、任务目标

### 2.1 业务目标

解决 P1-02 自动基线中信号4（A/B/C 三分类访谈规则）覆盖率不足的问题：
- 自动基线 auto_v1.0：信号4 仅覆盖 Excel 结构化数据（L4 组成/子产物），访谈规则标注"需人工继承"
- 手工基线 v1.0：包含 119 条完整访谈规则（A=47, B=31, C=41）
- **目标**：自动将手工基线规则合并到自动基线，生成完整基线 auto_v1.1

### 2.2 技术目标

1. 掌握 Markdown 表格解析（按节点分割 → 找信号4 标题 → 提取表格行）
2. 实现正则表达式精确匹配（避免贪婪匹配导致跨节点污染）
3. 验证合并后规则数量与手工基线完全一致

## 四、执行方案

### 4.1 架构

```
┌──────────────────────┐
│  手工基线 v1.0 (815行) │
│  9 节点 × 信号4 表格    │
└──────────┬───────────┘
           ↓
┌──────────────────────┐
│  ManualBaselineParser │
│  · 按 "### VN-XXX" 分割 │
│  · 找 "**信号4**" 标题   │
│  · 提取 | A/B/C类 | 行   │
└──────────┬───────────┘
           ↓
┌──────────────────────┐
│  BaselineMerger       │
│  · 替换整个信号4 区域   │
│  · 清除占位符+Excel规则  │
│  · 写入手工访谈规则      │
└──────────┬───────────┘
           ↓
┌──────────────────────┐
│  完整基线 auto_v1.1   │
│  760 行 (+50 行规则)   │
└──────────────────────┘
```

### 4.2 关键设计决策

| 决策 | 方案 | 原因 |
|------|------|------|
| 解析策略 | 字符串 `find()` + 局部正则 | 避免贪婪匹配跨节点 |
| 合并策略 | 替换整个信号4 区域（到信号5 前） | 清除自动基线中原有 Excel 提取规则，避免重复 |
| 验证策略 | 按节点统计（非全局 grep） | 避免表头/分隔线被误统计 |

### 4.3 关键发现

1. **手工基线实际规则数**：A=47, B=31, C=41 = 119 条（`grep -c "| A类 |"` 统计 48 包含了汇总表头行）
2. **自动基线信号4 已有内容**：Excel 提取的 L4 组成/子产物/物理对应规则（约 18 条），合并时需清除
3. **节点级匹配**：9 个节点全部命中，每个节点 A/B/C 数量与手工基线一致
