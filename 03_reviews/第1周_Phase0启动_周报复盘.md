## 三、遇到的问题与解决

| 问题 | 现象 | 根因 | 解决方案 | 经验教训 |
|------|------|------|---------|---------|
| GitHub HTTPS push 失败 | `LibreSSL SSL_connect: SSL_ERROR_SYSCALL in connection to github.com:443` | macOS 自带 LibreSSL 与 GitHub TLS 存在兼容性问题 | 改用 SSH 协议 | 遇到 SSL 错误时优先尝试 SSH |
| SSH 22 端口被封锁 | `Connection closed by remote host port 22` | 当前网络环境封锁 SSH 默认端口 | 在 `~/.ssh/config` 配置 `ssh.github.com:443` | 443 端口通常是防火墙放行端口，可作为 SSH fallback |

1. **工程化文档习惯**：每个文档开头加「文档定位说明」能显著降低后续维护成本。

3. **SSH 网络排错**：掌握了 `ssh-keygen`、GitHub SSH key 配置、以及 443 端口 fallback 的完整流程。
4. **决策日志价值**：把一次排错过程写成决策日志，下次遇到同类问题可以直接复用，而不是重新搜索。
